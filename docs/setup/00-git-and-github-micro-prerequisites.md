# Git and GitHub Micro Prerequisites

## Purpose

This file tells you the minimum you need before the course starts.

You do not need to be good at Git.

You do not need to know GitHub Actions already.

## What You Need for This Course

Before class, make sure you have:

- a GitHub account
- a web browser
- internet access
- the student repository link from your instructor

That is enough to start.

## What You Do Not Need at the Start

At the beginning of the course, you do not need:

- local Git commands
- Docker on your laptop
- cloud accounts

The early labs are browser-first on purpose.

## Small Words to Know Before Class

These five words are enough:

- repository: the project and its files
- commit: a saved change in GitHub
- workflow: an automation file
- Actions tab: where workflow runs appear
- log: the detailed output of a workflow step

## What Will Happen in the First Labs

In the first part of the course, you will:

1. create your own repository from the template
2. edit one file in the browser
3. commit the change in GitHub
4. open the `Actions` tab
5. read the workflow result

That is why local Git is not required at the start.

## If You Already Know Git

That is fine.

Still follow the course flow.

The goal is to learn the CI/CD story, not to race ahead with tools.
