# Help Guide

## Purpose

This folder is the student help and recovery area.

Open it when something feels unclear, a workflow fails, or you want one quick explanation without leaving the course story.

## Best Order When You Feel Stuck

1. [How to Read Actions Logs](01-how-to-read-actions-logs.md)
2. [Troubleshooting](02-troubleshooting.md)
3. [Glossary](03-glossary.md)

## Use These Pages By Need

- [Runner Mental Model](00-runner-mental-model.md): use when you want to understand where the workflow is running
- [How to Read Actions Logs](01-how-to-read-actions-logs.md): use when a workflow step failed
- [Troubleshooting](02-troubleshooting.md): use when you want the fastest likely recovery path
- [Glossary](03-glossary.md): use when a course term feels unfamiliar
- [YAML Cheatsheet for GitHub Actions](04-yaml-cheatsheet.md): use when YAML structure is the problem
- [Runner Types](05-runner-types.md): use later if you want more context about hosted versus self-hosted runners
- [Trigger Reference](06-trigger-reference.md): use when you need to compare trigger choices
- [Finding and Reusing GitHub Actions](07-finding-and-reusing-actions.md): use when a lab or exercise asks you to add an action

## One Reminder

When you are stuck, return to the course story:

1. code
2. verify
3. package
4. deliver

Then ask:

Which part of the story am I in right now?
