# Day 2 Opening Bridge

## Purpose

Use this short page at the start of Day 2.

It reconnects yesterday's CI ideas to today's packaging and delivery ideas.

## What We Learned on Day 1

Yesterday we focused on one question:

`How do we know a change is still safe?`

We learned how workflows, jobs, steps, triggers, runners, and tests help answer that question.

## Why CI Is Not Enough

Passing tests matters.

But passing tests does not tell us:

- what exact package we should carry forward
- whether we are delivering the same thing we already checked

## What Day 2 Adds

Today we continue the story:

1. verify
2. package
3. deliver

The new question is:

`What do we deliver after the code is verified?`

## Keep This Story in Mind

We are not changing the story.

We are extending it:

`code -> verify -> package -> deliver`
